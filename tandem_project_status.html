החזון: AI Product OS

המערכת נועדה לעזור למפתחים ויזמים עצמאיים לעבור מרעיון למוצר עובד, באמצעות מתודולוגיית SPDD (Structured Prompt-Driven Development). המערכת שומרת "זיכרון" של הפרויקט, מפיקה פרומפטים מובנים (Mission Briefs) עבור כלי AI (כמו Cursor או Claude), וקולטת את התוצרים כדי לנהל מעקב מבוסס ראיות.

יעדים מרכזיים (MVP)

  • הפיכת רעיון לדרישות מובנות (PRD)
  • שמירת "זיכרון" קבלת החלטות
  • ייצור משימות טכניות
  • שילוב שקוף של כלי AI חיצוניים

מחוץ לתחום (Non-goals)

  • תחליף מלא ל-IDE או Cursor
  • הרצת סוכנים אוטונומיים אוטומטית
  • ניהול הרשאות Enterprise מסובך

התקדמות הפרויקט

MVP v0.1 ✅ הושלם ועלה לפרודקשן. כעת בביצוע v0.2 — Artifact Flexibility.

פירוט שלבי הפיתוח

מה כבר נעשה — כל 8 הפאזות הושלמו ✅

  • Phase 0–1 — תשתית ו-Auth: Next.js 16 App Router, Tailwind v4, shadcn/ui, Supabase Auth + RLS, 10 טבלאות, route protection.
  • Phase 2 — Wizard + Brief: 3-step wizard (Idea → Clarify → Brief), AI service layer עם Zod schemas, generate-clarification + generate-brief routes.
  • Phase 3 — PRD + Architecture: PRD Generator, Architecture Generator, auto-save decisions + risks, Decision Log page.
  • Phase 4 — Tasks + Missions: Task Generator, Mission Brief Generator, MissionBriefCard עם copy button, Missions pages.
  • Phase 5 — Return Brief Loop: Return analysis AI, Drift Detection, Risk Register, computeNextAction לכל השלבים.
  • Phase 6 — Living Docs + Deploy: Markdown export, Landing Page, production build ✅ (31 routes), Vercel live — tandem-claude.vercel.app.
  • Phase 7 — AI Companion: floating chat panel, streaming API עם project context, platform routing (Cursor / Claude Code / v0 / Lovable), mission prompt + return prompt generation, return summary ingestion → שמירה ל-DB.
  • Smoke Tests (כל ה-SPDD loop): Signup → Wizard → Brief → PRD → Architecture → Tasks → Mission → Return → Companion — אומתו בפרודקשן ✅
  • Schema cleanup: database.sql הישן נמחק — migrations הם ה-source of truth ✅

v0.2 — בפיתוח עכשיו (2026-05-26)

  • Artifact Flexibility Migration: עמודות חדשות ל-artifacts (source, skip_reason, completeness_score) + טבלת artifact_versions עם היסטוריה מלאה + RLS.
  • ArtifactHealthPanel: ויזואליזציה של איכות Mission Brief — ציון %, סטטוס לכל ארטיפקט (complete / draft / missing / skipped), כפתור "דלג" עם סיבה.
  • EditableArtifact: עריכה מוטבעת של ארטיפקטים ישירות מהדף.
  • ImportArtifactDialog: ייבוא ארטיפקט קיים מ-Notion / Google Docs / טקסט חופשי.
  • EntryPointStep (Wizard): שלב חדש ראשוני ב-Wizard — "איפה אתה עכשיו?" עם 4 נתיבי כניסה: רעיון / PRD קיים / קוד קיים / כבר בבנייה.
  • i18n / Language Provider: תשתית רב-לשונית — lib/i18n/ + components/language-provider.tsx.
  • Artifact API Routes: app/api/artifacts/ (skip endpoint) + app/api/ai/artifacts/ (AI-powered operations).
  • נדרש: הרצת Migration 03 ב-Supabase SQL Editor + smoke test של EntryPointStep + Health Panel.

הערימה הטכנולוגית (Tech Stack)

  • Frontend: Next.js App Router, Tailwind CSS, shadcn/ui
  • Backend: Next.js Route Handlers, Supabase (PostgreSQL)
  • AI Layer: Vercel AI SDK (OpenAI / Claude)
  • Authentication: Supabase Auth + RLS
  • Deployment: Vercel

מודל הנתונים (Data Model)

  • Projects: מחזיק את המידע הראשי.
  • Artifacts: שומר גרסאות של PRD, Architecture, Brief (באמצעות JSONB).
  • Tasks: רשימת משימות מסודרת לסטטוסים.
  • Decisions: יומן החלטות המקושר ישירות ל-Artifacts ומשימות.
